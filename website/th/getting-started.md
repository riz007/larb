# เริ่มต้นใช้งาน

> ⚠️ **สถานะ: `0.1.0-alpha`** สำหรับนักพัฒนาที่ใช้คีย์ของตัวเองในช่วงแรกเท่านั้น
> API และคอนฟิกอาจเปลี่ยนได้ Larb ทำงาน **อัตโนมัติ** และรันคำสั่งบนเครื่องของคุณ
> — โปรดอ่าน[แบบจำลองความปลอดภัย](/th/security)ก่อนนำไปพึ่งพาจริง

## 1. ติดตั้ง

Larb ต้องใช้ **Node.js ≥ 20** ในช่วงอัลฟ่าจะเผยแพร่ภายใต้แท็ก `alpha` จึงติดตั้ง
ด้วย `@alpha`:

```bash
npm i -g @larb/cli@alpha
larb version
```

## 2. เลือกโมเดล (ใช้คีย์ของคุณเอง)

Larb ไม่ผูกกับโมเดลใด — ไม่มีผู้ให้บริการรายใดได้สิทธิพิเศษ เลือกด้วย `kind` ใน
`~/.larb/config.toml` แล้ว export คีย์ API ของผู้ให้บริการนั้น คุณเป็นเจ้าของคีย์
(อ่านจากสภาพแวดล้อม ไม่ใช่จากคอนฟิกในรีโพ) และเส้นทางการใช้งานเอง

```toml
# ~/.larb/config.toml
[provider]
kind = "openai"      # anthropic · openai · ollama · deepseek · gemini · groq ·
                     # mistral · xai · openrouter · together · perplexity

[provider.models]    # ไม่บังคับ — ละไว้เพื่อใช้ค่าเริ่มต้นของผู้ให้บริการ
orchestrator = "gpt-4o"        # โมเดลแรง: วางแผนและขับเคลื่อนลูป
worker       = "gpt-4o-mini"   # โมเดลถูก/เร็ว: งานย่อยและการบีบอัด
```

```bash
export OPENAI_API_KEY=sk-...    # ตัวแปรสภาพแวดล้อมของผู้ให้บริการที่คุณเลือก
larb providers                  # แสดงผู้ให้บริการ + คีย์ที่ตั้งไว้แล้ว
```

**อยากใช้ฟรี/ในเครื่อง?** ตั้ง `kind = "ollama"` แล้วรันโมเดลในเครื่อง — ไม่มีคีย์
ไม่มีค่าใช้จ่าย:

```toml
[provider]
kind = "ollama"
[provider.models]
orchestrator = "llama3.1"
```

## 3. การรันครั้งแรก

`larb ask` เป็นแบบ **อ่านอย่างเดียว** — วิธีที่ปลอดภัยในการยืนยันว่าทุกอย่างทำงาน:

```bash
larb ask "โปรเจกต์นี้ทำอะไร?"
```

`larb run` จะแก้ไขไฟล์ ครั้งแรกที่ชี้ Larb ไปยังไดเรกทอรี มันจะถาม **การตัดสินใจ
เชื่อถือ** (ไม่มีการอ่านคอนฟิกที่เป็นโค้ดหรือส่งข้อมูลผ่านเครือข่ายก่อนคุณตัดสินใจ)
จากนั้นทุกการเขียนไฟล์จะแสดง diff และทุกคำสั่งจะขออนุมัติ:

```bash
larb run "เพิ่มแฟลก --version ให้กับ CLI"
```

งาน `run` ยังไม่ "เสร็จ" จนกว่าคำสั่งตรวจสอบที่คุณตั้งไว้จะผ่าน:

```toml
verify = ["npm run -s typecheck", "npm test"]
```

## 4. สิ่งที่ควรเข้าใจก่อนนำไปพึ่งพาจริง

Larb เป็นเอเจนต์ที่ยึด **ความปลอดภัยมาก่อน** — สิ่งเหล่านี้เปิดใช้งานโดยค่าเริ่มต้น:

- **เชื่อถือก่อนทำสิ่งใด** — ไม่อ่านคอนฟิกที่เป็นโค้ดและไม่เรียกเครือข่ายจนกว่าคุณ
  จะตัดสินใจเชื่อถือไดเรกทอรี
- **สิทธิ์ตามความสามารถ** — การเขียนไฟล์ คำสั่ง เครือข่าย และการเรียก MCP ล้วน
  ต้องอนุมัติ (ครั้งเดียว / เซสชัน / เสมอ / ปฏิเสธ) และ **บันทึกทุกการอนุมัติ** ดู
  ได้ด้วย `larb audit`
- **เพดานค่าใช้จ่ายเด็ดขาด** — บัญชีโทเคน/ดอลลาร์แบบสดหยุดเอเจนต์ก่อนใช้จ่ายเกิน
  ปรับได้ใต้ `[limits]`
- **แซนด์บ็อกซ์การรันจริง** — คำสั่งรันในคอนเทนเนอร์ rootless (docker/podman) โดย
  กันความลับของโฮสต์ไว้และปิดเครือข่ายโดยค่าเริ่มต้น หากไม่มีรันไทม์จะถอยไปใช้
  ซับโพรเซสบนโฮสต์ที่แยกตัวน้อยลง และบอกระดับที่ใช้งานตอนเริ่มทุกการรัน

อ่าน**[แบบจำลองความปลอดภัย](/th/security)**ฉบับเต็มก่อนใช้งานแบบไม่มีคนดูแล

## 5. ก้าวต่อไป

- **คำแนะนำของโปรเจกต์** — วางไฟล์ `AGENTS.md` ในรีโพเพื่อบอก Larb ถึงคำสั่ง
  build/test และแนวปฏิบัติของคุณ (เชิงแนะนำ — ลบล้างสิทธิ์ไม่ได้)
- **เครื่องมือภายนอก (MCP)** — เพิ่มเซิร์ฟเวอร์ `[[mcp]]` ในคอนฟิกระดับโกลบอล แล้ว
  ตรวจดูด้วย `larb mcp` / `larb mcp probe` ดูเพิ่มที่[สถาปัตยกรรม](/th/architecture)
- **สกิล** — ติดตั้งส่วนขยายที่ควบคุมและเซ็นรับรอง: `larb skill <cmd>`
- **รันถาวร** — `larb runs` แสดงสแน็ปช็อต; `larb resume` ทำงานต่อจากที่ค้างไว้
